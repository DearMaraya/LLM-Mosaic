# LLM Mosaic

> **汇集，不编译。** 一种用大语言模型对同一主题的多份独立来源做并列呈现、差异标注、逐条溯源的知识组织模式。

## 这是什么？

LLM Wiki 回答的问题是"关于 X，我知道什么"。  
LLM Mosaic 回答的问题是**"关于 X，A 怎么说、B 怎么说、C 怎么说——以及他们哪里说得不一样"**。

当你对同一主题有多份独立来源——不同作者的传记、不同机构的研报、不同立场的法律文书——这些来源之间的**分歧本身就是最有价值的信息**。但传统的 LLM 工作流恰恰通过"综合"抹平了这些分歧。

LLM Mosaic 反其道而行之：**多源并列、逐条溯源。不做综合，让读者自己判断。**

## 核心原则

1. **汇集，不编译** — 把每份来源的说法一字排开，保持原貌，不合并，不替读者判断
2. **逐条溯源** — 每条说法后面跟一个可点击的链接，跳回原文精确位置
3. **一个实体一个页面** — 每个人物、事件、概念一个页面，汇集所有来源视角
4. **矩阵是总控台** — 实体×来源矩阵（`entity-index.md`）让 LLM 在编译前知道去哪找，只读该读的部分
5. **诚实标注空源** — 来源在这个实体上无话可说，显式标注"未专门述及"。一个来源选择不写什么，本身就是信息
6. **加一列，不重写** — 新增来源=矩阵加一列+选择性回刷，永远不需要全量重写

## 架构

```
原始资料（只读） → Mosaic 页面（多源并列、逐条溯源） → 产出层（可选）
```

两个辅助文件：`entity-index.md`（实体×来源矩阵）和 `log.md`（只追加操作日志）。

## 适用场景

任何"多个独立视角看同一个东西"的领域：

- 历史人物研究（多本传记互证）
- 法律判例分析（控方、辩方、判决书）
- 竞品情报（多份行业研报对照）
- 文学批评（不同批评流派解读）
- 多国视角的历史事件

## 起源

LLM Mosaic 的方法论源于苏东坡研究项目的实践——四本传记 + 三册年谱，400W 字，56 个实体 × 7 份来源的跨源矩阵。
