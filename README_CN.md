# LLM Mosaic

> **汇集，不编译。** 一种用大语言模型对同一主题的多份独立来源做并列呈现、差异标注、逐条溯源的知识组织模式。

---

## 这是什么？

2026 年 4 月，Karpathy 在 X 上提出了 LLM Wiki——让 LLM 增量构建一个持久化的 markdown 知识库。这是一个漂亮的洞察：知识应该沉淀、积累、复利增长。

但 LLM Wiki 和整个 RAG 生态共享一个盲区：**当多份独立来源对同一事物给出不同判断时，"综合"操作本身就是在抹掉最有价值的信息。**

林语堂说王安石是偏执狂，李一冰说他是理想主义改革家。LLM 综合成"王安石是一个复杂的历史人物"——这句话没错，但它抹掉了一切让你真正理解这个人的张力。

**LLM Mosaic 做相反的事：把每份来源的说法一字排开，让你自己看、自己判断。每条说法可点回原文的精确位置。分歧不被抹平，差异就是信息。**

## 🛠 工具链

```
Obsidian（IDE） + Claude Code（引擎） + Claudian（Agent）
```

- **Obsidian**：wikilink 跳转、图谱视图、双栏浏览
- **Claude Code**：像程序员操作代码库一样操作知识库
- **Claudian**：住在 Obsidian 里的 Agent，理解 wikilink 和 frontmatter
