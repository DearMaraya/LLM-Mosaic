# LLM Mosaic

> **汇集，不编译。**
>
> 当所有人都在用 LLM 做"综合"的时候，我们选择做相反的事——把多份独立来源对同一事物的说法一字排开，让分歧可见，让每句话可溯源。
>
> 这不是一个工具。这是一种**看待知识的方式**。

## 站在巨人的肩膀上，看见他们没看的地方

2025 年，Karpathy 提出了 LLM Wiki——让 LLM 增量地构建和维护一个持久化的 markdown 知识库。三层架构（原始资料→Wiki→Schema），三种操作（Ingest/Query/Lint）。这是一个漂亮的洞察：**知识不应该每次查询都从零推导，它应该沉淀、积累、复利增长。**

而传统 RAG 的生态里，NotebookLM、ChatGPT 文件上传、各类"给 AI 喂文档"的产品——核心模式都是"检索→综合→生成答案"。快、准、省事。

这两个方向共享同一个底层假设：**LLM 的工作是帮你消化信息，输出一个干净的答案。**

这个假设在大多数场景下是对的。但在一个场景下，它是错的——

**当你有多份独立来源对同一事物给出不同判断的时候，你要的不是"综合"，而是"全貌"。**

---

## 问题

LLM 擅长综合。给它五份报告，它给你一份"总体来看"。给它四本传记，它给你一个"复杂而多面"的人物画像。

这听起来很好。但有一个根本性的问题——

**"综合"的前提是：存在一个客观的真实版本，你只需要把它提炼出来。**

历史不是这样的。判例不是这样的。市场判断不是这样的。文学批评更不是。

林语堂说王安石是偏执狂，李一冰说他是理想主义改革家。LLM 综合成"王安石是一个复杂的历史人物"——这句话没错，但它抹掉了所有让你真正理解这个人的张力。差异被压缩了，视角被平均了，最值钱的信息被当成噪声扔掉了。

**Karpathy 的 LLM Wiki 是对的：知识应该沉淀。RAG 也是对的：检索能加速获取。但它们都没回答一个问题：当多个来源互相矛盾的时候，沉淀什么？检索什么？**

---

## 解法

LLM Mosaic 的核心原则只有一条：**汇集，不编译。**

| 主流做法 | LLM Mosaic |
|---|---|
| 读五份报告 → 给一个"综合结论" | 读五份报告 → 把五种判断并排放在你面前 |
| LLM 替读者判断 | 读者看原文自己判断 |
| 追求共识 | 保留分歧，分歧就是信息 |
| 每次查询重新推导 | 一次汇编，持续叠加新来源 |

```
❌ "综合来看，乌台诗案是北宋党争的产物……"  ← LLM 替读者做了判断

✅ "李一冰说：沈括是始作俑者。📖"
   "林语堂说：王安石的小人集团主导。📖"
   "周文翰说：印刷文化造就的文化明星被攻击。📖"
   "祝勇说：苦难造就艺术的前奏。📖"
                                            ← 读者看原文自己判断
```

这不是"不信任 AI"。这是承认一个简单的事实：**有些问题，答案的统一不如视角的并存更有价值。**

---

## 不是什么，是什么

**LLM Mosaic 不是 RAG。** RAG 是"检索相关片段→生成答案"。Mosaic 是"定位全部相关段落→逐源摘出→并列呈现"。RAG 追求答案，Mosaic 追求全貌。

**LLM Mosaic 不是 LLM Wiki。** LLM Wiki 是一个知识库，它的默认操作是综合。LLM Mosaic 是一种编译模式，它的唯一操作是汇集。你可以在 LLM Wiki 的三层架构里用 Mosaic 的方法编译页面——苏东坡项目就是这么做的。

**LLM Mosaic 是一种知识组织原语。** 就像 MapReduce 是分布式计算的原语、Transformer 是序列建模的原语——Mosaic 是多源互证场景下的知识编译原语。

---

## 怎么运作

三个动作：

**1. 建矩阵。** 新来源入库时，不直接读全文，先做实体扫描——这个来源提了哪些人物/事件/概念？分别在哪些章节？写入一张 `entity-index.md` 矩阵。

```
| 实体   | 李一冰 | 林语堂 | 周文翰 | 祝勇 | 年谱  |
|--------|--------|--------|--------|------|-------|
| 沈括   | 大量   | --     | 多次   | 少量 | 7条   |
| 王安石 | 大量   | 大量   | 少量   | 大量 | 23条  |
```

矩阵的价值：编译前知道去哪找，编译时只读该读的段落，新增来源时知道该补哪些实体。

**2. 编译页面。** 查矩阵→只读相关段落→按来源汇列原文→写差异表→诚实标注空源。

**3. 加一列，不重写。** 新增一本传记 = 矩阵加一列 + 选择性回刷高价值实体。永远不需要全量重写。随着来源从 3 份增长到 30 份，Mosaic 的价值不是"多了几条信息"，而是多了一个又一个独立的维度。

## 为什么是现在

这和模型能力无关。GPT-4 可以综合，GPT-5 也可以综合，GPT-6 还是可以综合。综合能力只会越来越强。

**但这个模式需要的恰恰不是"更强的综合"，而是"抑制综合的纪律"。**

随着 LLM 的上下文窗口越来越大、多文档处理越来越无损，一个反直觉的趋势正在浮现：综合成本趋近于零的时候，**差异反而变成了最稀缺的资源**。当任何人都能让 AI 吐出"王安石是一个复杂的历史人物"，能把四本传记并排摊开、每条引用精确到段落的人，拥有的是另一种东西——**不可被平均化的判断力素材。**

---

## 起源

LLM Mosaic 源于苏东坡研究项目——四本传记、三册年谱、400W 字原始资料、56 个实体 × 7 份来源的跨源矩阵。但它不限于历史。任何"多个独立视角看同一个东西"的领域——判例分析、竞品情报、文学批评、多国档案——都是 Mosaic 的适用场域。
